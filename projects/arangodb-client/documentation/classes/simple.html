    near

    (
    • [collection]
    • latitude
    • longitude
    • [options]
    )
    Promise

    Defined in lib/api/simple.js:294

    The default will find at most 100 documents near a given coordinate. The returned list is sorted according to the distance, with the nearest document coming first. If there are near documents of equal distance, documents are chosen randomly from this set until the limit is reached. In order to use the near operator, a geo index must be defined for the collection. This index also defines which attribute holds the coordinates for the document. If you have more then one geo-spatial index, you can use the geo field to select a particular index.

    Parameters:

    • [collection] String optional
      • The collection containing the documents.If not provided the collection defined in the connection is used.
    • latitude Number
      • The latitude of the coordinate.
    • longitude Number
      • The longitude of the coordinate.
    • [options] Object optional
      • JSONObject with optional parameters:
      • [skip] Number optional
        • can also be set using the "skip" method in this class.
      • [limit] Number optional
        • can also be set using the "limit" method in this class
      • [geo] Number optional
        • If given, the identifier of the geo-index to use.
      • [distance] Number optional
        • If given, the attribute key used to store the distance.
